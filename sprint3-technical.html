Use an analogy to describe the differences between HTML and CSS

HTML is the basic structure of a webpage. It holds the basic rules, plans for the page and is also the space where you link other resource materials. Think of HTML like a builder - working off plans and creating a strong structure that someone can inhabit down the line.

CSS is the styling of a webpage. In your CSS (or stylesheet file) is where you create the look for your webpage. The code for your CSS can also be written within your HTML but for confusion sake it is best to keep a seperate file and link the two together. This keeps your work tidy and allows you to know exactly where to go for those visual changes. Think of CSS like a decorator. The would help a builder or homeowner with interior styling, design, color schemes and the overall vibe of a space.

Explain control flow and loops using an example process from everyday life

Control flow is the order in which the computer completes tasks as written in the code. For example, in your daily routine you would have a way in which you execute your morning, drive to work, complete your work day and then drive home. There is a controlled flow in the start to finish of your day.

The computer will run code based on the order in which it is written from the beginning of the file to the end, much like how you will run your day from start to finish. But what happens when you are late or expected to take a detour on your way to work? This would be the equivalent of a ‘loop’ in your daily schedule. Let’s imagine you’re in your car driving down your normal route but then discover the road is closed up ahead. In order to complete your task of “getting to work” you will need to take a detour first to get to work. This will require you to take a couple extra left and right turns to get you back on your way to the office. Loops request the computer to run or perform a calculation and then give an answer before completing the rest of the code. If you don’t take the detour and get back on the main road, you ain’t going to get to work!

What is the DOM? How might you interact with it?

DOM = Document Object Model

Document - a file, let’s say your HTML file for your website

Object - anything inside your HTML file - tags, links, basically anything written within the HTML file

Model - how your HTML file is laid out. What parent tags do you have? How many sibling tags are there? How have you laid out text and images?

You can interact with your DOM from a text editor or directly in your chrome browser using ‘inspect’. If you are interacting with your file through a text editor, you will need to save your changes and then refresh your webpage to see the changes. If you are interacting with your file directly in your browser, you will see changes happen in real time. You do have the option to save changes you have made directly whilst using inspect in your browser but we will get into that later.

What is the difference between accessing data from arrays vs. objects?

Arrays are ordered collections of data. They can hold different types of data such as numbers and strings. When accessing data within an array, you will need to use an index number for specific items. Here is an example of an array:

var myArray = [1, 2, 3, 4]

var myArray = [“Sauce”, “Pepperoni”, “Mushroom”, “Mozzarella”]

If I wanted to call “Pepperoni” I would need to do the following:

console.log[1]

This would populate the answer “Pepperoni”. Even though pepperoni is the 2nd value in the array, computers count starting at 0. This means that the index numbers of the items within the array are as follows [0, 1, 2, 3]. If we wanted to call mushroom we would use the index number of [2].

In order to access data from an object however you will need to use a key “.” after your function has been called.

var myCabin = { Material = ‘timber’; Location = ‘lake’; Rooms = [‘living room’; ‘dining room’; ‘bedroom 1’; ‘bathroom’; ‘deck’] Age = 30 }

myCabin.age = 30 // this would show the age of the cabin located within the object

myCabin.rooms[3] = ‘bathroom’ // this would show the ‘bathroom’ string within the array that is in the object

Tell me, what are functions and why are they useful?

Functions ask your code to perform a task or make a calculation. These can be written as a single piece of code tucked nicely away within a block of code, and then called upon whenever you need it's super powers by using a single short command. Functions can save you time and finger typing when you are writing code for any project. You can also use this function multiple times so you do not have to continuously type out your code. Pretty neat eh?
